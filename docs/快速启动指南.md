#### 方法1: 使用vcpkg (推荐)

**安装vcpkg**:

```bash
# 克隆vcpkg仓库
```

**安装Eigen3**:

```bash

C:\vcpkg\vcpkg install eigen3:x64-windows

# 验证安装
```

## 🔧 第三步: 编译C++求解器

### 3.1 使用CMake配置项目

#### 方式A: 命令行配置 (推荐)

**打开"Developer Command Prompt for VS 2022"**:

```bash
# 进入项目根目录
cd C:\path\to\CAE_FEM_SOFTWARE

# 配置CMake (使用vcpkg)
cmake -S . -B build ^
    -G "Visual Studio 17 2022" ^
    -A x64 ^
    -DCMAKE_TOOLCHAIN_FILE=C:/vcpkg/scripts/buildsystems/vcpkg.cmake

# 或者手动指定Eigen路径
cmake -S . -B build ^
    -G "Visual Studio 17 2022" ^
    -A x64 ^
    -DEigen3_DIR=C:/Libraries/eigen3/share/eigen3/cmake
```

**配置成功标志**:

```
-- Build files have been written to: C:/path/to/CAE_FEM_SOFTWARE/build
```

#### 方式B: CMake GUI配置

1. 打开 CMake GUI (`cmake-gui`)
2. **Where is the source code**: `C:/path/to/CAE_FEM_SOFTWARE`
3. **Where to build the binaries**: `C:/path/to/CAE_FEM_SOFTWARE/build`
4. 点击 **Configure** → 选择 **Visual Studio 17 2022** → 平台选 **x64**
5. 如果出现Eigen3错误,添加条目:
   - Name: `Eigen3_DIR`
   - Type: `PATH`
   - Value: `C:/Libraries/eigen3/share/eigen3/cmake`
6. 点击 **Generate**

### 3.2 编译项目

**命令行编译** (推荐):

```bash
# Release版本 (用于正式使用)
cmake --build build --config Release

# Debug版本 (用于调试)
cmake --build build --config Debug

# 清理并重新编译
cmake --build build --config Release --clean-first
```

**VS2022 IDE编译**:

1. 打开 `build/CAE_FEM_SOFTWARE.sln`
2. 右键 `fem_core` 项目 → 生成
3. 确认配置为 `Release` 和 `x64`

### 3.3 验证编译结果

编译成功后,应该看到:

```
C:\path\to\CAE_FEM_SOFTWARE\
└── python\
    └── fem_core.cp39-win_amd64.pyd  ← 编译生成的Python模块
```

### 6.2 开发前处理界面

**场景**: 添加材料属性编辑对话框

1. **创建新文件** (`python/app/dialogs/material_dialog.py`):

```python
from PyQt6.QtWidgets import (QDialog, QVBoxLayout, QFormLayout,
                              QLineEdit, QPushButton, QLabel)

class MaterialDialog(QDialog):
    def __init__(self, parent=None):
        super().__init__(parent)
        self.setWindowTitle("材料属性编辑")
        self.init_ui()
    
    def init_ui(self):
        layout = QVBoxLayout()
        form = QFormLayout()
        
        self.name_edit = QLineEdit()
        self.E_edit = QLineEdit()
        self.nu_edit = QLineEdit()
        self.rho_edit = QLineEdit()
        
        form.addRow("材料名称:", self.name_edit)
        form.addRow("杨氏模量 (Pa):", self.E_edit)
        form.addRow("泊松比:", self.nu_edit)
        form.addRow("密度 (kg/m³):", self.rho_edit)
        
        layout.addLayout(form)
        
        btn_ok = QPushButton("确定")
        btn_ok.clicked.connect(self.accept)
        layout.addWidget(btn_ok)
        
        self.setLayout(layout)
```

2. **集成到主窗口** (`python/app/main_window.py`):

```python
from dialogs.material_dialog import MaterialDialog

class CAEMainWindow(QMainWindow):
    def create_material(self):
        dialog = MaterialDialog(self)
        if dialog.exec() == QDialog.DialogCode.Accepted:
            # 获取材料参数
            E = float(dialog.E_edit.text())
            nu = float(dialog.nu_edit.text())
            # 创建材料
            mat = fem_core.Material(1, E, nu)
            print(f"材料创建成功: E={E}, nu={nu}")
```

## 🐛 故障排查完整指南

### 问题诊断流程图

```
编译/运行失败
    ↓
检查Python版本匹配?
    ├─ 否 → 重新安装匹配版本的Python
    └─ 是 ↓
检查CMake配置成功?
    ├─ 否 → 检查Eigen3路径、pybind11安装
    └─ 是 ↓
检查.pyd文件存在?
    ├─ 否 → 重新编译 (cmake --build build --config Release)
    └─ 是 ↓
检查import fem_core成功?
    ├─ 否 → 检查Python路径 (sys.path)
    └─ 是 ↓
运行测试通过?
    ├─ 否 → 检查具体错误信息
    └─ 是 → 成功! 🎉
```

### 常见错误详解

#### 错误1: CMake找不到Python

**错误信息**:
```
CMake Error: Could not find PythonLibs
```

**解决方案**:
```bash
# 方法1: 指定Python可执行文件
cmake -S . -B build -DPYTHON_EXECUTABLE=C:/path/to/python.exe

# 方法2: 设置环境变量
set PYTHON_EXECUTABLE=C:\Anaconda3\envs\cae_env\python.exe
cmake -S . -B build
```

#### 错误2: pybind11未找到

**错误信息**:
```
CMake Error at CMakeLists.txt:10 (find_package):
  Could not find a package configuration file provided by "pybind11"
```

**解决方案**:
```bash
# 确认pybind11已安装
pip show pybind11

# 手动指定pybind11路径
cmake -S . -B build -Dpybind11_DIR="C:/path/to/site-packages/pybind11/share/cmake/pybind11"
```

#### 错误3: Eigen3找不到

**错误信息**:
```
CMake Error: Could not find Eigen3
```

**解决方案**:
```bash
# 方法1: 使用vcpkg
vcpkg install eigen3:x64-windows
cmake -S . -B build -DCMAKE_TOOLCHAIN_FILE=C:/vcpkg/scripts/buildsystems/vcpkg.cmake

# 方法2: 手动指定
cmake -S . -B build -DEigen3_DIR=C:/Libraries/eigen3/share/eigen3/cmake
```

#### 错误4: 编译错误 - 无法解析的外部符号

**错误信息**:
```
LNK2019: 无法解析的外部符号 "public: ..."
```

**可能原因与解决**:
1. **未包含所有源文件**: 检查 `CMakeLists.txt` 中的 `add_library` 是否包含所有 `.cpp` 文件
2. **编译器架构不匹配**: 确认使用 x64 架构编译
3. **Eigen库路径错误**: 确认 Eigen 正确链接

**调试步骤**:
```bash
# 清理build目录
rm -rf build

# 重新配置
cmake -S . -B build -G "Visual Studio 17 2022" -A x64

# 详细编译输出
cmake --build build --config Release --verbose
```

#### 错误5: 运行时DLL缺失

**错误信息**:
```
ImportError: DLL load failed while importing fem_core
```

**解决方案**:
```bash
# 检查Python版本匹配
python --version  # 必须与编译时一致

# 检查依赖DLL
# 使用Dependency Walker或dumpbin工具
dumpbin /DEPENDENTS python/fem_core.cp39-win_amd64.pyd

# 确保VS运行时库已安装
# 下载: Microsoft Visual C++ Redistributable for Visual Studio 2022
```

#### 错误6: PyQt6界面无法显示

**错误信息**:
```
qt.qpa.plugin: Could not find the Qt platform plugin "windows"
```

**解决方案**:
```bash

pip uninstall PyQt6 PyQt6-Qt6 PyQt6-sip
pip install PyQt6

# 方法2: 设置Qt插件路径
set QT_QPA_PLATFORM_PLUGIN_PATH=C:\path\to\site-packages\PyQt6\Qt6\plugins

# 方法3: 检查Qt安装
python -c "from PyQt6.QtWidgets import QApplication; app = QApplication([]); print('PyQt6正常')"
```

## 📚 进阶配置

### 配置1: 使用Intel MKL加速

**安装MKL**:

```bash
# 通过conda安装(推荐)
conda install mkl mkl-include

# 或下载Intel oneAPI Math Kernel Library
# https://www.intel.com/content/www/us/en/developer/tools/oneapi/onemkl.html
```

**修改CMakeLists.txt**:

```cmake
# 查找MKL
find_package(MKL REQUIRED)

# 链接MKL
target_link_libraries(fem_core PRIVATE MKL::MKL)
```

### 配置2: 启用OpenMP多线程

**修改CMakeLists.txt**:

```cmake
# 查找OpenMP
find_package(OpenMP REQUIRED)

# 链接OpenMP
target_link_libraries(fem_core PRIVATE OpenMP::OpenMP_CXX)

# 设置编译选项
target_compile_options(fem_core PRIVATE 
    $<$<CXX_COMPILER_ID:MSVC>:/openmp>
    $<$<CXX_COMPILER_ID:GNU>:-fopenmp>
)
```

**在代码中使用**:

```cpp
#include <omp.h>

// 在fem_solver.cpp中
void FEMSolver::assembleGlobalStiffnessMatrix() {
    #pragma omp parallel for
    for (size_t i = 0; i < elements.size(); i++) {
        // 并行组装单元刚度矩阵
    }
}
```

### 配置3: 生成优化的Release版本
